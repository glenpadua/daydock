# Security Policy

## Supported versions

Security fixes are applied to the latest `main` branch.

## Reporting a vulnerability

Please do not open public issues for potential vulnerabilities.

Report privately with:
- vulnerability description,
- impact assessment,
- reproduction steps or proof-of-concept,
- suggested mitigation (if available).

If private reporting infrastructure is not yet configured for this fork, contact the maintainer directly and request a private channel before sharing details.

## Response expectations

- Initial acknowledgement target: within 72 hours
- Triage target: within 7 days
- Patch timeline: depends on severity and complexity
